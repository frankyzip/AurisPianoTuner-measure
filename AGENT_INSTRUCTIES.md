# Agent Communicatie Instructies

## Taalvoorkeur
**BELANGRIJK**: Alle communicatie met gebruikers moet in het Nederlands plaatsvinden.

## Communicatierichtlijnen

### 1. Algemene Taalregels
- Gebruik altijd Nederlandse taal in alle antwoorden
- Spreek de gebruiker aan met "je" (informeel) of "u" (formeel) afhankelijk van de context
- Gebruik Nederlandse technische terminologie waar mogelijk
- Bij internationale begrippen: gebruik de Nederlandse vertaling met eventueel de Engelse term tussen haakjes

### 2. Technische Communicatie
- Variabelnamen en code blijven in het Engels (zoals gebruikelijk in programmering)
- Commentaar in code moet in het Nederlands
- Foutmeldingen en uitleg moeten in het Nederlands
- Documentatie en instructies altijd in het Nederlands

### 3. Voorbeelden van Correcte Communicatie

#### ? Goed:
```
"Ik ga een nieuwe knop toevoegen aan het formulier. Deze knop zal de meting starten."
"De ASIO driver (audio stuurprogramma) wordt geïnitialiseerd."
"Er is een fout opgetreden bij het laden van het bestand."
```

#### ? Fout:
```
"I will add a new button to the form."
"The ASIO driver is being initialized."
"An error occurred while loading the file."
```

### 5. Uitzonderingen
- Officiële namen van frameworks, bibliotheken en technologieën blijven Engels (.NET, Windows Forms, etc.)
- Code snippets en programmeertaal keywords blijven Engels
- Foutcodes en systeemmeldingen mogen Engels blijven indien dit de standaard is

### 6. Tone of Voice
- Professioneel maar vriendelijk
- Helpend en ondersteunend
- Duidelijk en begrijpelijk
- Geduldig bij uitleg van complexe concepten

## Implementatie
Elke agent die werkt aan dit project moet deze richtlijnen volgen voor consistente Nederlandse communicatie met alle gebruikers.
